Приложение: трекер диеты

Цель: отслеживать параметры питания по дням

Параметры питания:
1. есть 3 типа питания:
 - 🥩 мясо
 - 🧀 животные продукты без мяса
 - 🥦 только растительная пища
2. есть 3 типа объёма:
 - мало пищи
 - умеренное количество
 - обильная еда
3. наличие вредной пищи:
 - да
 - нет
4. алкоголь:
 - да
 - нет

Стек: JS, HTML, PWA, хранение данных в Local Storage

Ключевые фичи:
1. внесение данных по питанию раз в день
	- на фронте форма для их внесения находится в отдельном блоке
	- этот блок отображается , только если за текущую дату данные ещё не внесены
	- для внесения данных нужно заполнить 5 параметров:
		-- дата (предзаполнена)
		-- тип питания
		-- объём пищи
		-- наличие вредной пищи
		-- наличие алкоголя
	- обязательным параметром является только дата
	- есть кнопка "Записать"
	- результаты сохраняются в Local Storage
	- после записи блок скрывается
2. возможность редактирования данных за любой день
	- если на графике (см. ниже) выбран какой-либо день, то отображается блок редактирования данных
	- используется та же самая форма, что и для добавления данных
	- параметры предзаполняются значениями, которые уже сохранены
	- при сохранении данные за выбранный день перезаписываются
	- после сохранения блок редактирования скрывается
3. отображение графиков
	- наверху страницы есть график, в котором отображаются сохранённые данные
	- блок с графиком отображается всегда
	- этот график выглядит как табличка 7x5
	- подгружаются данные за последние 28 дней
	- одновременно видны только 7 столбцов
	- можно скроллить таблицу по горизонтали
	- те 7 дней, которые на экране после скролла, считаются текущим периодом и используются в статистике (см. ниже)
	- в верхней строке подписаны даты
	- в остальных строках отображаются параметры, если они сохранены, либо прочерк
	- тип питания отображается в виде эмодзи: 🥩 для мяса, 🧀 для животных продуктов без мяса, 🥦 для растительной пищи
	- объём пищи отображается в виде эмодзи: 🔻 для малого, ничего для умеренного, 🔺 для повышенного
	- наличие вредной пищи отображается в виде эмодзи: 💔 если да, ничего если нет
	- наличие алкоголя отображается в виде эмодзи: 🍺 если да, ничего если нет
	- если на какую-либо дату сохранённых данных нет, то заполнена только верхняя строка с самой датой
	- при нажатии на столбец этот столбец считается выбранным
	- он помечается яркой рамкой
	- выбор нового столбца снимает выбор со старого
	- повторное нажатие на тот же столбец снимает с него выбор
	- при смене выбора подгружается блок статистики, а также блок редактирования
4. отображение статистики
	- под графиком есть блок статистики
	- правила статистики:
		-- 2 дня алкоголя подряд - 🔴 предупреждение
		-- 2 дня алкоголя за текущий период - 🟠 предупреждение
		-- 3 дня вредной пищи за текущий период - 🟠 предупреждение
		-- малый объём пищи даёт 1 очко нестабильности, повышенный - тоже 1 очко
		-- если за текущий период 2 и более очка нестабильности, то 🟠 предупреждение
		-- если за текущий период 4 и более очка нестабильности, то 🔴 предупреждение
		-- 2 дня нестабильности подряд - 🔴 предупреждение
		-- если предупреждений нет, то отображается надпись: "🟢 Всё в порядке"
5. пожелания к интерфейсу
	- единственная страница
	- порядок блоков:
		-- график
		-- статистика
		-- внесение/редактирование данных
	- оформление тёмное
	- оформление лаконичное
	- ориентация страницы - вертикальная
	- возможен только вертикальный скролл
	- блоки растянуты по ширине
	- в блоке редактирования поля - это переключатели между соответствующими вариантами
	- варианты помечаются как ээмодзи
